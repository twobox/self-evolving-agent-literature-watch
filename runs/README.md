# Run Index

本文件是所有运行报告的规范索引。历史报告不可改写；每次新运行只在下表追加一行，并更新 YAML 中的 `last_updated`。

## 运行报告命名

运行报告路径使用：

```text
runs/YYYY/YYYY-MM-DD-NN.md
```

其中：

- `NN` 是同一天从 `01` 开始的两位序号；
- Run ID 等于不含 `.md` 的文件名，例如 `2026-06-26-01`。

## 更新规则

- 每次完整运行必须追加一行，即使没有发现有效候选。
- 统计数字必须与对应运行报告一致。
- `报告` 使用指向仓库内 Markdown 文件的相对链接。
- 新增首条记录时删除“当前尚无运行记录”，但保留本节标题和表头。
- Run ID、报告文件名以及 `STATE.md` 中记录的最近一次成功运行 ID 必须一致。
- 不得为同一次运行创建多个索引行，也不得另建并行运行索引。
