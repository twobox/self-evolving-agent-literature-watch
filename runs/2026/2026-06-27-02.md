## 2. 本次主要发现

Harness 演化已经从“调一个 Prompt”扩展为同时修改指令、技能、工具和工作流代码。RHO 只使用过去轨迹和自偏好即可生成 Harness 更新；Self-Harness 从模型自身失败模式出发提出最小修改并通过回归测试筛选；Meta-Harness 则把历史候选的源码、得分和轨迹作为外循环搜索状态。

Agent Skills 方向开始把技能视为可训练的外部状态。SkillOpt 使用受控增删改操作、拒绝缓冲区和验证集门控优化单个 Skill 文档；SkillOS 用强化学习训练 SkillRepo 的长期整理策略；SkillHone 进一步保留诊断、修改、证据和结果组成的决策历史，以支持跨会话继续演化。

更广义的 Agent 代码演化形成了从 STOP、Gödel Agent、ADAS 到 Darwin Gödel Machine 的连续路线：演化对象从脚手架函数扩展到完整 Agent 代码、工具和控制流，并通过档案或经验保留成功设计。Voyager 虽超出三年滚动窗口约一个月，但其可执行技能库和环境反馈驱动的技能修正仍是理解当前 Agent Skills 工作的重要基础。
