# Self-Evolving Agent Literature Watch

本仓库用于发现可能相关且值得后续阅读的 Self-Evolving LLM Agent 论文，并维护这些论文的基本信息。

仓库只做候选论文收集与客观信息维护，不负责论文精读，也不在监控阶段评价方法创新性、实验充分性、证据可信度或论文质量等级。

## 三种独立动作

仓库提供三个相互独立的运行入口：

1. **近期论文监控**：发现最近公开的相关论文，重点补充首次公开或发表状态以及作者机构情况；
2. **历史优质论文搜索**：围绕指定研究方向补充历史论文，重点参考正式发表情况和引用情况；
3. **已有候选信息更新**：只核查 `candidates.md` 中已有论文，直接更新可确认的公开或发表信息。

一次运行只执行用户指定的一种动作，不默认同时执行其他动作。

## 快速开始

### 近期论文监控

```text
请在当前仓库执行一次近期论文监控。先读取 AGENTS.md，并按照仓库规则更新候选池、运行报告、运行状态和运行索引。
```

### 历史优质论文搜索

```text
请在当前仓库搜索 [研究方向] 的历史优质论文。先读取 AGENTS.md，只执行历史论文搜索，不扫描近期论文，也不更新已有候选信息。
```

### 已有候选信息更新

```text
请在当前仓库更新已有候选论文的信息。先读取 AGENTS.md，只核查 candidates.md 中论文的公开和发表状态，不发现新论文。
```

更多可复制提示词见 [`PROMPTS.md`](PROMPTS.md)。

## 仓库结构

| 文件或目录 | 作用 |
| --- | --- |
| [`AGENTS.md`](AGENTS.md) | Agent 的任务入口、动作边界和写入要求 |
| [`GUIDE.md`](GUIDE.md) | 三种运行方式、相关性判断和信息记录规则 |
| [`PROMPTS.md`](PROMPTS.md) | 可直接复制的运行提示词 |
| [`STATE.md`](STATE.md) | 三种动作各自的最近运行和实际覆盖记录 |
| [`candidates.md`](candidates.md) | 唯一候选论文池，包含用户手动备注列 |
| [`queries.md`](queries.md) | 静态研究方向和检索词参考 |
| [`RUN_REPORT_TEMPLATE.md`](RUN_REPORT_TEMPLATE.md) | 轻量运行报告结构 |
| [`runs/README.md`](runs/README.md) | 历次运行报告索引 |

## 核心原则

- 只根据标题和摘要判断论文是否可能与目标主题相关；
- 近期论文记录作者机构等背景信息，但不打分、不分级；
- 历史论文记录正式发表和引用等客观信息，但不建立质量评价系统；
- 不维护推荐、暂缓、忽略、待复核等论文状态；
- `candidates.md` 是唯一候选池，同一论文只保留一条记录；
- `用户备注` 完全由用户维护，Agent 不得修改；
- 每次运行只完成一种动作，并留下简短、可追溯的运行记录。
