# 可复制的运行提示词

仓库提供三种独立动作。每次提示词只指定一种动作，不要求 Agent 同时执行其他搜索或维护任务。

## 1. 近期论文监控

```text
请在当前仓库执行一次近期论文监控。

先读取 AGENTS.md，并按照 GUIDE.md、STATE.md 和 queries.md 确定本次时间范围与查询。只根据标题和摘要判断主题相关性，补充首次公开或发表状态以及作者机构情况。

只发现近期新论文，不搜索历史论文，不查询引用情况，不更新已有候选信息。完成后将新增论文写入 candidates.md，创建简短运行报告，并更新对应状态和运行索引。必须保留用户备注列，不得填写或修改用户备注。
```

需要指定时间范围时：

```text
请在当前仓库执行一次近期论文监控，时间范围为 [开始日期] 至 [结束日期]。其他要求按照 AGENTS.md 和 GUIDE.md 执行。
```

## 2. 历史优质论文搜索

```text
请在当前仓库搜索 [研究方向] 的历史优质论文。

先读取 AGENTS.md，并按照 GUIDE.md、STATE.md 和 queries.md 执行。根据标题和摘要判断主题相关性，重点参考正式发表情况和引用情况，记录本次实际搜索方向、时间或来源范围。

只执行历史论文搜索，不扫描近期论文，不调查作者机构，不更新已有候选信息，也不建立待复核或质量分级系统。完成后将新增论文写入 candidates.md，创建简短运行报告，并更新对应状态和运行索引。必须保留用户备注列，不得填写或修改用户备注。
```

可指定更具体的范围，例如：

- Agent Memory 与 Experience Reuse；
- Harness、Prompt 与 Agent Skills；
- Agent Reinforcement Learning；
- 轨迹、工作流与工具使用学习；
- 自我改进评测与能力边界；
- 某个会议、期刊、年份或核心论文的引用网络。

## 3. 已有候选信息更新

```text
请在当前仓库更新已有候选论文的信息。

先读取 AGENTS.md、GUIDE.md、STATE.md 和 candidates.md。只核查 candidates.md 中已有论文的首次公开时间、正式发表状态、论文页面、标题和必要的机构信息，并直接更新表格对应单元格。

不要发现或加入新论文，不重新判断相关性，不刷新引用情况，不为论文增加“已更新”“无变化”或其他状态。运行报告只列出实际发生的信息变化。必须原样保留所有用户备注。
```

需要限定范围时：

```text
请只更新 candidates.md 中 [指定论文、序号范围或条件] 的公开和发表信息，其他要求按照 AGENTS.md 和 GUIDE.md 执行。
```

## 4. 只讨论方案，不修改仓库

```text
请根据当前仓库规则讨论 [近期论文监控 / 历史优质论文搜索 / 已有候选信息更新] 的执行方案，但不要修改任何仓库文件。
```

## 5. 提示词原则

- 一次只指定一种动作；
- 可以补充时间、研究方向、会议、期刊或论文范围；
- 不必重复表格字段和判断规则，固定要求以 `AGENTS.md` 与 `GUIDE.md` 为准；
- 不使用“完整运行”指代同时完成三种动作。
