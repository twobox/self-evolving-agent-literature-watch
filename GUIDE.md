# Self-Evolving LLM Agent 论文候选收集指南

## 1. 目标

发现与 Self-Evolving LLM Agent 及相邻方向相关、值得后续阅读的论文，并维护候选论文的基本信息。

相关方向包括（只要属于基于大语言模型的Agent的自我进化即可）：

- 自进化LLM Agent
- Agent 经验积累与复用；
- 记忆更新与长期记忆；
- Agent Skills、Harness、Prompt 与上下文优化；
- 轨迹、工作流与工具使用学习；
- Agent 强化学习与持续学习；
- 自我改进及相关评测。

根据论文标题和摘要判断主题相关性。

## 2. 运行类型

每次运行选择一种类型（如果用户没说，需要询问）：

1. 近期论文监控；
2. 历史优质论文搜索；
3. 已有候选信息更新。

## 3. 近期论文监控

### 目标

发现近期新公开的相关论文。

### 可选来源

- arXiv；
- OpenReview；
- 主要会议近期接收列表；
- 作者、实验室或机构页面。

### 记录内容

- 论文标题与链接；
- 一句话介绍；
- 首次公开日期和当前发表状态（例如ICML 2025）；
- 作者机构（例如清华；腾讯）。

用户未指定时间范围时，从 `STATE.md` 中上次覆盖日期之后开始；尚无记录时可选择3个月以内的。

## 4. 历史优质论文搜索

### 目标

围绕指定研究方向、时间段、会议、期刊或来源，补充历史论文。

### 参考来源

- arXiv
- OpenReview
- 会议和期刊论文列表；
- 引用数据库；
- 综述和代表性论文的参考文献；
- 核心论文的被引论文、相似论文；
- 代表性作者的相关工作。

### 记录内容

- 论文标题与链接；
- 一句话介绍；
- 首次公开或正式发表情况；
- 引用数量。（当时查到什么就写什么即可，不需要精确）

## 5. 已有候选信息更新

核查 `candidates.md` 中已有论文，更新：

- 首次公开时间；
- 正式发表会议或期刊；
- 论文链接；
- 标题；
- 作者机构。
- 引用数量。（当时查到什么就写什么即可，不需要精确）

运行报告记录发生变化的论文及更新前后信息。

## 6. 候选论文总表

`candidates.md` 使用以下表格：

| 序号 | 论文标题 | 一句话介绍 | 首次公开或发表状态 | 作者机构情况 | 引用 | 用户备注 |
| :--: | --- | --- | --- | --- | --- | --- |

字段要求：

- `论文标题` 使用完整标题并链接到可靠论文页面（如果无法链接也无所谓）；
- `一句话介绍` 根据标题和摘要概括；一般是针对什么问题，提出什么，怎么做的，有啥贡献。
- `首次公开或发表状态` 如果是首次公开，可直接记录 首次公开；如果是已被收录的，则记录收录情况，例如ICML 2025。
- `作者机构情况` 使用简短机构名称；
- `引用`直接记录当前查到的即可，没查到就留空。
- `用户备注` 由用户维护，Agent 更新表格时原样保留。

同一论文只保留一条记录，按规范化标题去重。

## 7. 运行报告

每次运行新建：

```text
runs/YYYY/YYYY-MM-DD-NN.md
```

参考报告模版。

## 8. 执行步骤

1. 确认运行类型；
2. 读取规则、候选表、状态、检索词和报告模板；
3. 执行搜索或信息核查；
4. 更新 `candidates.md`；
5. 新建运行报告；
6. 更新 `STATE.md`；
7. 更新 `runs/README.md`；
