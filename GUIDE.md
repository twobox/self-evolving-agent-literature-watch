# Self-Evolving LLM Agent 论文候选收集指南

## 1. 目标

发现与 Self-Evolving LLM Agent 及相邻方向相关、值得后续阅读的论文，并维护候选论文的基本信息。

相关方向包括：

- Agent 经验积累与复用；
- 记忆更新与长期记忆；
- Agent Skills、Harness、Prompt 与上下文优化；
- 轨迹、工作流与工具使用学习；
- Agent 强化学习与持续学习；
- 自我改进及相关评测。

根据论文标题和摘要判断主题相关性。

## 2. 运行类型

每次运行选择一种类型：

1. 近期论文监控；
2. 历史优质论文搜索；
3. 已有候选信息更新。

## 3. 近期论文监控

### 目标

发现指定时间范围内新公开的相关论文。

### 来源

- arXiv；
- OpenReview；
- 主要会议近期接收列表；
- 作者、实验室或机构页面。

### 记录内容

- 论文标题与链接；
- 一句话介绍；
- 首次公开日期和当前发表状态；
- 作者机构。

用户未指定时间范围时，从 `STATE.md` 中上次覆盖日期之后开始；尚无记录时选择一个有限的近期范围。

## 4. 历史优质论文搜索

### 目标

围绕指定研究方向、时间段、会议、期刊或来源，补充历史论文。

### 来源

- 会议和期刊论文列表；
- 引用数据库；
- 综述和代表性论文的参考文献；
- 核心论文的被引论文、相似论文；
- 代表性作者的相关工作。

### 记录内容

- 论文标题与链接；
- 一句话介绍；
- 首次公开或正式发表情况；
- 引用情况。

引用信息注明来源和查询日期，例如：

```text
NeurIPS 2024；引用 320（Semantic Scholar，2026-06-27）
```

## 5. 已有候选信息更新

核查 `candidates.md` 中已有论文，更新：

- 首次公开时间；
- 正式发表会议或期刊；
- 论文链接；
- 标题；
- 作者机构。

运行报告记录发生变化的论文及更新前后信息。

## 6. 候选论文总表

`candidates.md` 使用以下表格：

| 序号 | 论文标题 | 一句话介绍 | 首次公开或发表状态 | 作者机构情况 | 用户备注 |
| ---: | --- | --- | --- | --- | --- |

字段要求：

- `论文标题` 使用完整标题并链接到可靠论文页面；
- `一句话介绍` 根据标题和摘要概括；
- `首次公开或发表状态` 记录公开日期、发表信息，历史论文可同时记录引用情况；
- `作者机构情况` 使用简短机构名称；
- `用户备注` 由用户维护，Agent 更新表格时原样保留。

同一论文只保留一条记录，按规范化标题去重。

## 7. 运行报告

每次运行新建：

```text
runs/YYYY/YYYY-MM-DD-NN.md
```

报告记录：

- Run ID、运行类型、日期和实际范围；
- 查询与来源；
- 新增论文或信息变化；
- 新增或更新数量。

## 8. 执行步骤

1. 确认运行类型；
2. 读取规则、候选表、状态、检索词和报告模板；
3. 执行搜索或信息核查；
4. 更新 `candidates.md`；
5. 新建运行报告；
6. 更新 `STATE.md`；
7. 更新 `runs/README.md`；
8. 核对 Run ID、路径和数量。
