# Agent Instructions

本仓库用于发现可能相关且值得后续阅读的 Self-Evolving LLM Agent 论文，并维护候选论文的基本信息。

本仓库不是论文精读或质量评审系统。Agent 不负责评价方法创新性、实验质量、代表性或证据可信度。

## 任务入口

执行任务前，先判断用户要求的是哪一种动作：

1. 近期论文监控；
2. 历史优质论文搜索；
3. 已有候选信息更新。

三种动作相互独立。一次运行只执行用户指定的一种动作，不顺带执行其他动作。

所有动作至少读取：

1. `GUIDE.md`：运行规则；
2. `candidates.md`：现有候选和去重参照；
3. `STATE.md`：对应动作的最近运行与覆盖记录；
4. `queries.md`：检索方向与关键词参考；
5. `RUN_REPORT_TEMPLATE.md`：轻量运行报告格式。

## 共同判断边界

- 只根据论文标题和摘要判断是否与监控主题可能相关；
- 不要求确认论文是否真正实现持久化、自我演化或能力提升；
- 不深入检查方法、实验、消融或结论；
- 不给论文打分、分级或添加推荐、暂缓、忽略、待复核等状态；
- 不保存普通搜索噪声或被排除论文清单。

## 三种动作

### 1. 近期论文监控

只负责：

- 搜索用户指定时间范围内近期公开的论文；
- 根据标题和摘要判断主题相关性；
- 记录首次公开或当前发表状态；
- 记录作者机构情况；
- 将新增论文写入 `candidates.md`；
- 新建简短运行报告，并更新对应状态和运行索引。

不得查询引用情况，不得批量更新已有候选信息，不得执行历史补漏。

### 2. 历史优质论文搜索

只负责：

- 围绕用户指定的研究方向、时间、会议、期刊或来源搜索历史论文；
- 根据标题和摘要判断主题相关性；
- 参考正式发表情况和引用情况寻找较有影响力的论文；
- 将新增论文写入 `candidates.md`；
- 记录本次实际搜索方向和范围；
- 新建简短运行报告，并更新对应状态和运行索引。

不得扫描近期论文，不得调查作者机构，不得更新已有候选信息。

### 3. 已有候选信息更新

只负责：

- 核查 `candidates.md` 中已有论文；
- 更新可确认的首次公开时间、正式发表 venue、标题或机构等基本信息；
- 直接修改对应表格单元格；
- 在运行报告中只列出实际发生的更新；
- 更新对应状态和运行索引。

不得发现或加入新论文，不得重新判断论文相关性，不得为每篇论文增加更新状态分类。

## 候选池约束

- `candidates.md` 是唯一候选论文池；
- 同一论文只保留一条记录，通常按规范化标题去重；
- 论文标题应链接到 arXiv、OpenReview、会议页面或正式论文页面；
- `一句话介绍` 只根据标题和摘要客观概括；
- `作者机构情况` 使用简短机构名称，例如“清华大学；腾讯”；
- `用户备注` 完全由用户维护，Agent 不得填写、改写、清空或移动其中内容；
- 更新已有行时必须保留原有用户备注。

## 运行落盘要求

每次实际执行一种动作时：

1. 在 `runs/YYYY/` 新建一份 `YYYY-MM-DD-NN.md` 报告；
2. 按该动作需要更新 `candidates.md`；
3. 只更新 `STATE.md` 中与本次动作对应的记录；
4. 在 `runs/README.md` 追加一条运行索引；
5. 核对 Run ID、报告路径、新增或更新数量一致。

即使本次没有新增论文或没有信息变化，也应生成简短报告，如实记录零结果。

## 工作边界

除非用户明确要求，本仓库不负责：

- 下载或保存论文 PDF；
- 生成完整论文笔记；
- 检查代码、模型或数据集；
- 建立论文质量评分系统；
- 刷新全库引用数据；
- 修改外部论文笔记仓库。
