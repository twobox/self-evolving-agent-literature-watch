# Agent Instructions

本仓库用于持续执行 Self-Evolving LLM Agent 相关论文的发现、筛选、去重和状态跟踪。

## 任务入口

执行论文监控任务前，应依次读取以下文件：

1. `GUIDE.md`：论文监控的规范要求。
2. `STATE.md`：当前仓库的运行状态。
3. `candidates.md`：当前有效论文候选条目，避免重复监控。
4. `SCREENING_LOG.md`：历史论文筛选记录，避免重复监控。
5. `queries.md`：检索方向与关键词池。
6. `RUN_REPORT_TEMPLATE.md`：论文监控运行报告模板。

## 完整运行要求

当用户要求要求执行一次监控时，按照 `GUIDE.md` 执行操作，不只在对话中返回结果，也不只新增运行报告。

完整运行至少包括：

- 新建 `runs/YYYY/YYYY-MM-DD-NN.md`；
- 更新 `candidates.md`，增加本次推荐的论文条目；
- 按需更新 `SCREENING_LOG.md`，增加本次筛选掉的论文条目；
- 更新 `STATE.md`；
- 更新 `runs/README.md`；
- 最后核对。

即使本轮没有发现有效候选，也必须生成运行报告并更新相关状态。

## 数据与状态约束

- 同一论文在 `candidates.md` 中只记录一条规范记录，一般按论文标题判断是否为同一篇论文。
- 不得把“本次首次发现的旧论文”描述为“近期新发布论文”。
- 不得回写历史运行报告；论文后续变化应记录在新的运行报告及候选总表中。
- “本次出现类型”和“仓库当前状态”必须使用不同字段。
- 只把容易重复出现、已实质判断排除的文章写入 `SCREENING_LOG.md`，不要保存全部检索噪声。
- `STATE.md` 和 `runs/README.md` 使用固定结构；更新时应保留既有标题、字段名和表格列，不得另建并行状态区或自定义索引格式。
- 不得重新创建 `ignored.md`、`templates/run-report.md` 或其他与规范文件并行的别名入口。

## 工作边界

除非用户明确要求，本仓库中的监控运行不负责：

- 下载或保存论文 PDF；
- 检查代码、模型和数据集；
- 生成完整论文笔记；
- 修改外部论文笔记仓库；
- 为达到数量要求而降低筛选标准。

如发现适合生成正式笔记的论文，只更新其候选状态和后续动作。
