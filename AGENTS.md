# Agent Instructions

本仓库用于收集与 Self-Evolving LLM Agent 相关的候选论文，并维护基本信息。

## 任务入口

执行任务前，先确认本次动作：

1. 近期论文监控；
2. 历史优质论文搜索；
3. 已有候选信息更新。

一次运行只执行一个动作。

开始前读取：

- `GUIDE.md`；
- `candidates.md`；
- `STATE.md`；
- `queries.md`；
- `RUN_REPORT_TEMPLATE.md`。

## 近期论文监控

- 按指定时间范围搜索近期论文；
- 根据标题和摘要判断主题相关性；
- 记录首次公开或发表状态、作者机构；
- 将新增论文写入 `candidates.md`；
- 创建运行报告，更新 `STATE.md` 和 `runs/README.md`。

## 历史优质论文搜索

- 按指定研究方向、时间、会议、期刊或来源搜索历史论文；
- 根据标题和摘要判断主题相关性；
- 记录发表和引用情况；
- 将新增论文写入 `candidates.md`；
- 创建运行报告，更新 `STATE.md` 和 `runs/README.md`。

## 已有候选信息更新

- 核查 `candidates.md` 中已有论文；
- 更新首次公开时间、正式发表状态、论文链接、标题或作者机构；
- 在运行报告中记录实际变化；
- 更新 `STATE.md` 和 `runs/README.md`。

## 候选表规则

- 同一论文只保留一条记录，按规范化标题去重；
- `一句话介绍` 根据标题和摘要客观概括；
- `用户备注` 由用户维护，Agent 不得修改；
- 更新已有行时保留原有用户备注。

## 运行记录

每次运行：

1. 在 `runs/YYYY/` 新建 `YYYY-MM-DD-NN.md`；
2. 按需更新 `candidates.md`；
3. 更新本次动作对应的状态；
4. 在 `runs/README.md` 追加索引；
5. 核对 Run ID、报告路径和数量。

没有新增或更新时，也生成一份简短报告。
