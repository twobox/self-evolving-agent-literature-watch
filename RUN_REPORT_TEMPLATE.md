# 论文候选收集运行报告模板

本模板用于三种独立动作的轻量运行报告。每次报告只保留与本次动作有关的内容，不创建推荐、暂缓、忽略、待复核或无变化列表。

## 1. 运行信息

- Run ID：
- 报告路径：
- 运行类型：近期论文监控 / 历史优质论文搜索 / 已有候选信息更新
- 运行日期：
- 实际范围：
- 实际查询：
- 实际来源：

Run ID 必须与文件名及 `runs/README.md` 一致。报告路径使用：

```text
runs/YYYY/YYYY-MM-DD-NN.md
```

## 2. 本次结果

根据运行类型只保留下面一种结构。

### 近期论文监控或历史优质论文搜索

| 论文 | 一句话介绍 | 首次公开或发表状态 | 作者机构情况 |
| --- | --- | --- | --- |

历史搜索没有调查作者机构时，该列留空即可；引用情况可简洁写入“首次公开或发表状态”。

### 已有候选信息更新

| 论文 | 更新前 | 更新后 |
| --- | --- | --- |

只记录实际发生的信息变化，不列出无变化论文。

## 3. 运行摘要

- 新增或更新数量：
- 候选池是否已同步：
- 对应状态是否已更新：

没有新增或没有信息变化时，直接写明本次结果为 0，不增加其他分类。
